## LinkedIn Post

---

**AI doesn't need better formatting instructions. It needs teachable models of human expertise.**

For years, we've been prompting AI to produce layouts:

"Use Heading 2."
"Add a table."
"Increase spacing."

These instructions tell AI *what* to produce. They don't teach it *why* a particular visual structure works.

I've been working on something that changes this.

**Visual Literacy** is a reasoning framework that teaches AI to make visual design decisions by modelling human perceptual psychology — not formatting rules.

The key insight: experienced designers don't calculate every spacing decision. They instantly recognize when a page feels balanced, crowded, or confusing. That intuition is built from years of experience.

AI doesn't possess this intuition. So we translated it into explicit reasoning principles:

→ 24 design heuristics grounded in Gestalt psychology
→ An 8-dimension evaluation rubric (score any design 0-5, /40 total)
→ A translation dictionary that converts "make it bigger" into the perceptual intent behind it
→ A conflict resolution framework for when design rules contradict

But here's what I think is actually significant:

The architecture we built is domain-agnostic.

The six layers — Mental Model → Perceptual Principles → Heuristics → Conflict Resolution → Evaluation Rubric → Translation Layer — could be applied to any field where experts operate on tacit knowledge.

How architects perceive buildings. How physicians perceive patients. How teachers perceive learning. How leaders perceive teams.

The challenge is the same in each case: **making expert intuition teachable.**

Visual Literacy is the first chapter.

If you work with AI-generated designs, documents, or interfaces — and you've ever thought "something is wrong but I can't name it" — this gives you the vocabulary.
